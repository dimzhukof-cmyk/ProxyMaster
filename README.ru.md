# ProxyMaster

> Прозрачный прокси-клиент для Windows 11/10 — аналог Proxifier.
> Направляет весь (или выборочный) сетевой трафик через SOCKS5 или HTTP прокси без настройки каждого приложения отдельно.

## Возможности

- **Весь трафик системы** — перехватывает все исходящие TCP-соединения на уровне ядра через [WinDivert](https://reqrypt.org/windivert.html)
- **Выбор приложений** — отметь галочкой нужные процессы из списка; только их трафик пойдёт через прокси
- **SOCKS5 и HTTP CONNECT** — полная поддержка обоих протоколов с авторизацией
- **Совместимость с браузерами** — в режиме «все приложения» устанавливает системный прокси Windows (WinInet), браузеры работают автоматически
- **Безопасное хранение пароля** — пароль шифруется через Windows DPAPI (ключ привязан к учётной записи пользователя), в файл настроек записывается только зашифрованная строка
- **Тёмный интерфейс** — WPF-приложение с живым логом событий и счётчиком трафика
- **Защита от SSRF** — HTTP CONNECT к loopback-адресам (127.x.x.x) блокируется с ответом `403 Forbidden`

## Установка

1. Скачай архив последнего релиза: `ProxyMaster-x.x.x-win-x64.zip`
2. Распакуй в любую папку
3. Правой кнопкой на `ProxyMaster.exe` → **Запуск от имени администратора**

> **Установщик не нужен.** Настройки сохраняются в `%AppData%\ProxyMaster\settings.json`.

## Использование

### Режим «Все приложения» (по умолчанию)
1. Введи адрес прокси-сервера, порт и учётные данные
2. Нажми **Сохранить**
3. Нажми **▶ ЗАПУСТИТЬ**

ProxyMaster устанавливает системный прокси Windows и активирует WinDivert — весь TCP-трафик идёт через прокси.

### Режим «Только выбранные»
1. Переключи на **Только выбранные**
2. Нажми **↻ Обновить список** — появятся все запущенные процессы
3. Отметь галочками нужные приложения (например, `chrome.exe`, `telegram.exe`)
4. Нажми **▶ ЗАПУСТИТЬ**

Только отмеченные приложения перехватываются. Системный прокси не устанавливается — остальные приложения работают напрямую.

### Остановка
Нажми **■ СТОП**. WinDivert выгружается, системный прокси сбрасывается.

## Как это работает

```
Приложение (Chrome) ──SYN──► WinDivert (ядро Windows)
                                    │
                        подменяет dst → 127.0.0.1:8877
                                    │
                        TransparentProxyServer (локальный TCP)
                                    │
                        SOCKS5 / HTTP CONNECT handshake
                                    │
                        Прокси-сервер (например, 1.2.3.4:1080)
                                    │
                              Интернет 🌐
```

- **WinDivert** перехватывает исходящие SYN-пакеты и переписывает адрес назначения на локальный порт
- **ConnectionTracker** хранит таблицу: исходный порт → оригинальный адрес назначения
- **TransparentProxyServer** находит оригинальный адрес по порту и туннелирует соединение через SOCKS5/HTTP
- **SystemProxy** устанавливает WinInet-прокси для браузеров в режиме «все приложения»

## Безопасность

| Угроза | Защита |
|---|---|
| Пароль в файле настроек | Шифрование через Windows DPAPI (`ProtectedData`, уровень текущего пользователя) |
| SSRF через HTTP CONNECT | Запросы `CONNECT` к `127.*`, `localhost`, `::1` блокируются с ответом `403 Forbidden` |
| Несанкционированный доступ к локальному прокси | Слушает только `127.0.0.1`; соединения не с loopback отклоняются |
| DoS / лавина соединений | Жёсткий лимит 200 одновременных соединений |

## Известные ограничения

- **Только TCP** — UDP-трафик не перехватывается (большинство приложений используют TCP)
- **Только IPv4** — перехват IPv6 не реализован
- **Нужны права администратора** — WinDivert не работает без повышенных привилегий
- **Антивирусные предупреждения** — некоторые антивирусы помечают WinDivert как подозрительный (это легитимный драйвер ядра, используемый во многих инструментах)
